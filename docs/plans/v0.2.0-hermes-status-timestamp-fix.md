# V0.2.0 Hermes 状态时间修复

## 目标

修复 Hermes 已同步新数据，但系统状态仍显示“17 天前”的问题。

## 根因

`usage_records.timestamp` 中同时存在 ISO 文本时间和毫秒整数时间。SQLite 对混合类型执行 `MAX(timestamp)` 时会优先选中文本值，导致状态接口忽略更新的整数时间。

## 实施范围

1. 数据库初始化时把可解析的历史文本时间迁移为毫秒整数。
2. 状态和健康接口显式按整数时间聚合，避免混合类型再次误判。
3. 备份现有数据库，重启服务后验证 Hermes 最新时间、整体最新时间与同步状态。

## 验收标准

- Hermes 最新记录显示为当天数据，而不是 17 天前。
- Hermes 健康状态与最新记录时间一致。
- 状态接口和健康接口均返回有效的最新时间。
- 项目构建通过，数据库完整性检查通过。
